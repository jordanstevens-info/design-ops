# Templates

Shared, non-skill artifact shapes. Copy a template, fill it in, and save the
result where its scope lives. Read this only when creating or validating an
artifact.

- `scope.md` - project working brief and control file.
- `idea.md` - lightweight parked possibility, not yet a task or plan.
- `log.md` - dated receipt for shareable Work OS changes.
- `feedback.md` - OS-level, agent, eval, or Work OS behavior correction.
- `received-feedback.md` - personal growth feedback (manager, peer, 360, review).
- `project-feedback.md` - stakeholder or project feedback near a scope.

Add a template only when an artifact shape repeats. Templates used by a single
skill live inside that skill folder instead.
