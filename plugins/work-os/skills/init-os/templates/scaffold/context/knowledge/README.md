# Knowledge

Tracked knowledge is reusable Work OS truth. Put personal goals, tasks, voice
samples, and personal references in `personal/`. Team/company/client workspace
state stays in root `workspaces/`.

## Buckets

- `work-os/` - context protocol, operating principles, graduation rules, and
  shareable system doctrine.

Add a new bucket only when multiple files need the same home. Otherwise start
with a clear file name in the closest existing bucket. Common buckets that get
earned over time include org/team source maps, design or domain practice, and
reusable research, plus a cross-cutting `tooling-profiles.md` for tool and MCP
defaults — create each only when real content arrives.
