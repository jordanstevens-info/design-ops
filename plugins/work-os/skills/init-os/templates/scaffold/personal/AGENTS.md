# Personal Work OS

Local layer for solo goals, tasks, side projects, voice, and references. Track
it for solo local commits. It can be ignored later for team distribution.

## Load Order

1. Root `AGENTS.md`.
2. This file.
3. Relevant personal goal, task, sideproject, voice, reference, or log files.

Load only what changes the current task.

## Personal Structure

These sub-folders are created when earned, not pre-scaffolded. Add one only when
real content needs a home.

- `profile.md` - user settings and stable preferences, when needed.
- `goals/` - personal strategic goals and career direction.
- `ideas.md` - lightweight personal ideas not yet promoted to goals, tasks,
  references, or feedback.
- `tasks/` - personal cross-workspace kanban.
- `feedback/` - received feedback, growth themes, and synthesis from manager,
  peer, 360, stakeholder, or self-review signals.
- `sideprojects/` - personal projects only. Team, company, and client work
  stays in root `workspaces/`.
- `voice/` - personal writing voice and samples.
- `references/` - private or personal source material, including raw HR or
  performance-review documents.
- `log/` - private dated receipts. Created when the first personal receipt is
  written.

## Maintenance

Keep `AGENTS.md` and `memory.md` under 4,500 characters. If this router grows
past the cap, extract durable truth to local knowledge, corrections to
feedback, or repeatable behavior to a skill.

Use `work-os-feedback` for received feedback intake and routing. Keep raw review
source material in `personal/references/hr/`; route synthesized growth themes to
`personal/feedback/` and only promote them to goals or tasks when action is
earned.
