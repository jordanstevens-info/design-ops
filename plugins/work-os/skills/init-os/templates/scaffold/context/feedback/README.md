# Feedback Index

Context feedback is for OS-level, agent, eval, and Work OS behavior
corrections. Read this index only when that feedback context matters. Do not
scan every feedback entry by default.

Personal manager, peer, 360, or review feedback belongs in
`personal/feedback/`. Project or stakeholder feedback belongs nearest the
project scope first.

## Lifecycle

Feedback statuses: `captured`, `proposed`, `active`, `applied`, `graduated`,
`archived`, `dropped`, `superseded`.

Applied does not automatically mean archived. Keep applied feedback indexed
while it is still load-bearing. Feedback graduates when another artifact owns
the behavior, and archives only when it is no longer active guidance.

Index sections mean:

- **Active Agent Guidance** - active or still-load-bearing applied feedback.
- **Pending Graduation** - captured or proposed feedback awaiting action.
- **Recently Applied** - temporary visibility after a change lands.
- **Graduated / Historical** - canonical behavior lives elsewhere now.
- **Dropped / Superseded** - intentionally inactive or replaced feedback.
