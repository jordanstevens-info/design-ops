# Output Contract

Use this when lineage is useful: persisted artifacts, packages, reports,
external destination updates, review/eval/system-performance work, or handoffs
that need source refs and authority fields.

Do not append this as a footer to inline drafts or copy-pasteable messages.
Inline Slack updates, Jira ticket drafts, and other paste-ready content should
stay clean. If lineage is needed for an inline draft, keep it outside the
copy-paste body as a separate note only when useful.

Tiny accepted-only work can use a short inline summary, but still name the stop
condition and next action when that helps the user.
