# /pd-engineer

Work in the real codebase on a design change.

## Arguments

- `scope`: design target, Figma URL, ticket, branch goal, or component/page name

## Workflow

Route to `product-design-workflow` in engineer mode. Bootstrap `.design/` when needed, work on a `design/<topic>` branch, verify, write changelog, commit, and push the design branch when appropriate.

## Boundary

This command does not create or suggest MRs. Use `/pd-create-mr` explicitly after design branch work is complete.
