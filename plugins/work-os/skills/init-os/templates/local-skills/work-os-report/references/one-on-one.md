# 1:1 Doc — shape

Manager / direct-report 1:1 prep. Recurring entries per meeting; the doc
accumulates over time as a running log.

## Shape

```markdown
# <Manager> <> <Report> — 1:1 Doc

If you are stuck:
- Progress on Priorities — (e.g. What's happened since we last connected?)
- Goals — (e.g. What are your top goals for this coming week?)
- Challenges, barriers or blockers? — (e.g. What clarity can I provide? How
  can I assist to help move things forward?)
- Gratitude — (e.g. What went well? Who supported you? What are you
  thankful for?)
- Next steps — (What do you need to be successful, what are some actionable
  steps?)

---

Date: <date>
```

## Conventions

- **Well-being check-in** is a number 1–10. Optional. Don't fabricate one;
  ask if not supplied.
- **Last week / This week** beats vague paragraphs — concrete updates anchor
  the conversation.
- **Gratitude** lists people by name with what they helped with. Keeps the
  recurring section human, not perfunctory.
- **Next Steps** is numbered and short — actionable, not aspirational.
- Keep entries dated and append-only. Don't rewrite past entries; add a new
  one each cycle.

## When to use this shape

Use for **1:1** mode. Use for **personal** mode when the user wants 1:1-prep
output rather than a designer-status report.

For weekly designer status, see `designer-status.md`.
